    january 5, 2008 - Version 3.1

    Genral changes:
  • cartobase optionally uses the boost library. It is very little used yet, but is helpful to achieve thread-safety in smart pointers.
  • Improvements in generic objects (better genericity in IO, implemented and compiled on more types), fixed some linker problems caused by multiple instanciation of templates. Allow optional/missing builtin properties in PropertySet. Cleaned the code and removed obsolete implementations.
  • XML MINF 2.0 reading
  • Imrovements in smart pointers: thread-safety (using boost atomic counters), and some new types of veratile smart pointers for specific applications: weak_shared_ptr and shared_pointer
  • fixes in the portable threads API
  • improvements in generic Converter codes
  • python bindings for a large part of cartobase API: see the pyaims project
  • an important fix in Allocators when used to allocate/free arrays of classes with constructors/destructors (destructors were not called)
  • ... (a lot more)
